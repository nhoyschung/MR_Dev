# /export-pptx — PowerPoint Export

Triggers the **pptx-orchestrator** agent-team to produce a bilingual PPTX pair
(English master + Korean translation) for any MR-System report type.

## Usage

```
/export-pptx [type] [parameters]
```

| Type | Example | Output |
|------|---------|--------|
| `market-briefing` | `/export-pptx market-briefing HCMC 2025-H1` | 7-slide market briefing |
| `land-review` | `/export-pptx land-review "Binh Duong" "Thu Dau Mot" 2.5ha` | 12-slide land review |
| `project-profile` | `/export-pptx project-profile "Masteri Thao Dien"` | 5-slide project profile |
| `competitor` | `/export-pptx competitor "Masteri Thao Dien" "Vista Verde"` | 5-slide comparison |

## City Aliases

| Alias | Full Name |
|-------|-----------|
| HCMC, HCM | Ho Chi Minh City |
| BD | Binh Duong |
| HN | Hanoi |

## Output

Two `.pptx` files in `output/`:
- `{type}_{city}_{period}_{timestamp}_en.pptx` — English master
- `{type}_{city}_{period}_{timestamp}_ko.pptx` — Korean translation

Intermediate artifacts in `output/jobs/{job_id}/`:
- `raw_data.json` — DB query results
- `slide_content_en.json` — English slide content (content-writer authored)
- `slide_content_ko.json` — Korean translation

Manifest updated at `output/pptx_manifest.json`.

## Agent-Team Architecture

The export spawns a 6-agent team for quality:

```
Task 1 [data-analyst]    → DB queries → raw_data.json
Task 2 [content-writer]  → raw_data → slide_content_en.json  (expert English)
Task 3 [builder-en]  ┐   → slide_content_en.json → EN .pptx  (parallel)
Task 4 [ko-translator] ┘  → slide_content_en.json → slide_content_ko.json
Task 5 [builder-ko]      → slide_content_ko.json → KO .pptx
Task 6 [orchestrator]    → update manifest + MEMORY.md
```

Tasks 3 and 4 run in **parallel** after the content-writer finishes.

## Re-translation

To update the Korean translation without re-querying the DB:
```
/ko-translate {job_id}
```

## Implementation

Spawn the `pptx-orchestrator` agent with:
- `report_type`: derived from the `[type]` argument
- `params`: city, period, project names, or land_input dict as appropriate
- `job_id`: timestamp generated by orchestrator at startup
